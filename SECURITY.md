# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Use this section to tell people how to report a vulnerability.

*   Please do not report security vulnerabilities through public GitHub issues.
*   Please email `security@example.com` with a description of the issue.
*   You should receive a response within 24 hours.

## Security Best Practices
*   **No Real Money**: This app is for training only. Do not modify the code to accept real wagers.
*   **Data Privacy**: All data is stored locally (`localStorage`). No data is sent to external servers in this version.
*   **Dependencies**: We audit dependencies regularly with `npm audit`.
